01 Welcome to Noveller

Noveller is a feature-rich desktop writing application for Windows, built for fiction authors who want a Scrivener-style authoring environment with deep world-building tools. Write chapters, manage characters and locations, plan with Kanban boards and timelines, visualise story health, and export to PDF, EPUB, DOCX, or HTML — all without leaving the app.

This documentation hub is the starting point for everything you can customise: the template system that powers character sheets and world-building cards, the skin system that controls visual themes, and the community resources on GitHub.

New here? Start with the Template Authoring Guide — it covers the .weave format from the ground up, with worked examples for every field type and conditional rule.

03 Template System (.weave)

Noveller's character sheets, location cards, and item sheets are all driven by plain-text .weave template files. The syntax is designed to be readable and writable without any tooling — open a file, add fields, save, reload.

Templates support nine field types (text, multiline, number, tags, list, rate, dropdown, and entity @references), conditional visibility, template inheritance via Extends, and inline hint text. The parser assembles the UI automatically from top to bottom.

File location Place your .weave files in the Templates\ folder inside your Noveller data directory. Any file here appears as a selectable template inside the app.

Quick syntax overview

Every template follows a three-level hierarchy: #Category creates a tab, ##Section creates a collapsible group, and FieldName: [type] defines a field. Fields must live inside a section.

Tip Use Extends BaseName as the first line of a template to inherit all fields from another .weave file, then add or override sections below it. Great for genre-specific variants built on a shared core.

04 Skin System

Skins change the visual appearance of Noveller — colours, typography scale, editor background, and UI chrome — without touching any code. A skin is a small JSON file that maps named design tokens to values.

The built-in skin engine supports light and dark base modes, custom accent colours, paper textures, and font overrides for the editor and UI surfaces. Switching skins is instant and persists per-book.
