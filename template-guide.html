01 Document Hierarchy

Every template is built from three nested layers. Noveller reads them top-to-bottom and assembles the UI automatically.

// Level 1 — Category (tab)
#CategoryName

// Level 2 — Section (collapsible group)
##SectionName

// Level 3 — Field
Key Label: [type]
.weave
Rule Fields must live inside a ##Section. A category with no section won't render any fields.

Minimal working example

#Identity

##Basic
Name: [text]
Age: [number]
Species: [text]
.weave

02 File & Naming

Templates are plain text files saved with the .weave extension and placed in the Templates/ folder.

Templates/
  Overview.weave          ← auto-detected Overview tab
  BasicFantasy.weave
  DetailedFantasy.weave
  SciFiCrew.weave
folder
Tip A file named Overview.weave is automatically used as the character's Overview tab. It does not need an Extends statement and is the first tab shown.

03 Template Inheritance

A template can extend another, inheriting all its categories and sections then appending its own.

Extends BasicFantasy

#Magic

##Abilities
Primary Magic: [text]
Mana Pool: [number]
DetailedFantasy.weave
  • The Extends line must be the first non-blank line of the file.
  • Write only the base template's filename without extension — e.g. Extends BasicFantasy.
  • Inheritance is recursive: a chain of A → B → C works fine.
  • Merging order: base categories come first, child categories append after.
Warning The base file must exist in the same Templates/ folder or the Extends line is silently ignored.

04 Field Types Overview

Every field is declared with a key label, a colon, and a bracketed type expression:

Key Label: [type]
Key Label: [type=default,modifier,modifier2]
text
Single-line string input
multiline
Large paragraph textarea
number
Numeric input
tags
Chip-style keyword list
list
Ordered item list with optional cap
rate
Visual rating with range & style
dropdown
Select from enumerated options
@reference
Link to characters, items, or locations

05 text

A single-line string input. Accepts an optional default value.

Nickname: [text]
Title: [text=Novice]       ← default value "Novice"
SyntaxDescription
[text]Plain single-line input
[text=default]Pre-filled with a default string

06 multiline

A large free-form textarea, ideal for backstories, descriptions, or notes.

Backstory: [multiline]
Personal Notes: [multiline=Unknown origins.]

07 number

A numeric input. Accepts an optional numeric default.

Age: [number]
Starting Gold: [number=50]

08 tags

A chip-style list of freeform keywords. Useful for personality traits, skills, or categories. An optional second argument caps how many tags are allowed.

Personality Traits: [tags]
Primary Skills: [tags,5]     ← max 5 tags
SyntaxDescription
[tags]Unlimited tags
[tags,N]Maximum of N tags

09 list

An ordered list of items — each entry is a separate line (not a chip). Supports an optional item cap.

Known Spells: [list]
Top Allies: [list,3]          ← max 3 entries
SyntaxDescription
[list]Unlimited list items
[list,N]Maximum of N list items

10 rate

A visual rating control. Supports an optional default value, a numeric range, and a display style.

Leadership: [rate]                        ← default star, no range
Magic Skill: [rate=3,0-10]               ← starts at 3, range 0–10
Courage: [rate,1-5,heart]              ← hearts, range 1–5
Agility: [rate=2,0-5,circle]           ← circles, default 2
ArgumentPositionDescription
rate=N1stDefault starting value
min-max2nd (modifier)Numeric range, e.g. 0-10 or 1-5
star / heart / circle3rd (modifier2)Visual display style (default: star)
Note The range argument is only parsed when it contains a - dash. A plain number in the second position is treated as a maxItems cap (same as tags/list).

11 dropdown

A select field with a fixed list of options. Declared by providing comma-separated option values — no explicit type keyword needed. Noveller auto-detects this when the first token is not a known type name.

Alignment: [Lawful Good,Neutral,Chaotic Evil]
Class: [Warrior,Mage,Rogue,Cleric]
Rank: [Recruit,Soldier,Officer,Commander]
Warning — Name Collision If your first option is a reserved word (text, multiline, number, tags, list, rate), it will be parsed as that field type instead of a dropdown. Choose option names that don't clash with reserved keywords.

12 References

Reference fields link a character to other entities in your project. Prefixed with @.

Character references

Mentor: [@character]        ← single character
Allies: [@characters]       ← multiple characters
Rivals: [@characters,3]    ← multiple, max 3

Item references

Signature Weapon: [@item]
Artifacts: [@items]
Carried Items: [@items,6]

Location references

Birthplace: [@location]
Known Territories: [@locations]
Visited Regions: [@locations,5]
Type keywordLinks to
@characterOne character
@charactersMultiple characters (optional max)
@locationOne location
@locationsMultiple locations (optional max)
@itemOne item
@itemsMultiple items (optional max)

13 Conditionals

Any category, section, or field can be shown or hidden based on the value of another field.

Syntax

Element Declaration ? FieldKey=expectedValue

The element is only rendered when the named field's current value matches the expected value exactly.

Conditional category

#Magic ? UsesMagic=true

Conditional section

##Advanced Techniques ? MagicTier=Expert

Conditional field

Dragon Bond Name: [text] ? HasDragonBond=true

Full conditional workflow example

#Identity

##Basics
Name: [text]
UsesMagic: [Yes,No]           ← dropdown acting as toggle

#Magic ? UsesMagic=Yes        ← whole category hidden if No

##Abilities
Primary Magic: [text]
Mana Pool: [number]

##Advanced ? UsesMagic=Yes    ← section-level conditional
Arcane Theory: [multiline]
Note Conditions use exact string matching. UsesMagic=true and UsesMagic=True are different. Match the case of the dropdown option or text value precisely.

14 Field Hints

Add a placeholder hint text to any field by appending a quoted string after the closing bracket.

Backstory: [multiline] "Where did they come from?"
Nickname: [text] "What do allies call them?"
Power Level: [number] "Approx. 1–100"

Hints appear as grey placeholder text inside the input and disappear when the user starts typing.

15 Comments

Lines beginning with // are ignored by the parser and can be used for notes.

// This template is for high-fantasy campaigns
#Identity

// TODO: add Guild reference field later
##Basic
Name: [text]
