A complete guide to .skin files — file format, the full color token reference, and a worked example from minimal to complete.

01 What Is a Skin?

A skin is a JSON file that remaps every color in Noveller's UI. When Noveller loads a skin it reads your color definitions, converts them into brushes, and merges them into the application's resource dictionary — replacing the built-in theme without any code changes required.

Skins live alongside Noveller's built-in themes and can be selected from the same theme picker. Any file you drop into the Skins folder is discovered automatically the next time the picker opens.

Tip You only need to define the colors you want to override. Any token you omit falls back to Colors.Black (or Colors.Transparent for scrollbar backgrounds), so start minimal and add tokens until the UI looks right.

02 File & Location

Skin files use the .skin extension and must be placed in the Noveller Skins folder inside your Documents directory.

Documents/
  Noveller/
    Skins/
      MidnightInk.skin        ← appears in the theme picker
      WarmParchment.skin
      CoolSlate.skin
folder

The name shown in Noveller's theme picker is the filename without its extension — not the name field inside the JSON. MidnightInk.skin always appears as MidnightInk in the picker regardless of what you write in the file's name field.

Tip The Skins folder is created automatically the first time Noveller scans for custom themes, so you don't need to create it manually.

03 File Format

A skin file is valid JSON with a single top-level object containing a Colors dictionary. Each key is a color token name and each value is a color string.

{
  "Colors": {
    "Surface.Background": "#1e1e2e",
    "Surface.Border":     "#313244",
    "Text":               "#cdd6f4",
    "Accent":             "#cba6f7"
  }
}
.skin
  • The file must be valid JSON. Comments are supported — the parser explicitly ignores JSON comment syntax.
  • The Colors object must have at least one entry or the skin is rejected silently.
  • Keys are case-sensitive. Surface.Background is valid; surface.background is not recognised.
  • An unrecognised key is ignored. A bad color value logs a debug warning and is skipped.
Note — Color. prefix Keys may be written with or without a leading Color. prefix — both are equivalent. "Surface.Background" and "Color.Surface.Background" resolve to the same token. Prefer the shorter form without the prefix.

04 Metadata Fields

Alongside Colors, a skin file can carry four optional metadata fields at the top level. None of them affect how colors are loaded — they exist for display and tooling purposes.

{
  "name":        "Midnight Ink",
  "author":      "Jane Doe",
  "description": "A deep dark theme with violet accents.",
  "isDark":      true,
  "Colors": {
    // ...
  }
}
.skin
FieldTypePurpose
namestringHuman-readable display name. Not used by the theme picker — the picker always shows the filename without extension.
authorstringCredit for the skin creator. Shown in skin metadata UI where supported.
descriptionstringA short summary of the skin's style or intent.
isDarkbooleanHints to the UI that this is a dark theme. Used for thumbnail previews and other display decisions — the loader itself does not read this value.
Warning — Picker Name Renaming a skin in the picker requires renaming the .skin file itself. Changing the "name" field inside the JSON has no effect on the dropdown label.
Tip Even though isDark doesn't change how colors load, it's worth setting correctly. Noveller can use it to render an appropriate preview swatch in the theme picker, so users can tell at a glance whether a skin is light or dark before applying it.

05 Color Values

Any string that WPF's ColorConverter can parse is accepted. In practice that means hex notation and named colors.

FormatExampleNotes
#RRGGBB#1e1e2eSix-digit hex, most common
#AARRGGBB#cc1e1e2eEight-digit hex with alpha channel
#RGB#1e2Three-digit shorthand, no alpha
Named colorTransparent, Black, WhiteAny WPF/X11 named color
Warning — Alpha & Scrollbars Alpha values in hex colors work everywhere except the four scrollbar tokens, which have their opacity set programmatically and will ignore any alpha you provide. For scrollbars, use an opaque color and let Noveller apply the opacity.

07 Text

Text tokens cover every foreground label in the UI. At minimum, set Text — the others inherit from it where unset.

Text

Disabled / inactive labels
Text.Error

Text on offset / raised surfaces
Tip Input.Text is intentionally mapped to the same color as Text. You do not need to set a separate input text token — changing Text affects both.

11 Editor

Editor tokens control the writing canvas and its context menu. The editor is the largest visual surface in Noveller, so these tokens have a significant impact on the overall feel of a skin.

Editor.Background
Writing canvas background
Editor.Text
Prose text color (also used for editor menu text)
Editor.Selection
Selected text highlight
Editor.Menu.Background
Editor context menu background and its hover state
Note Editor.Menu.Background is intentionally applied to both the resting and hover state of the editor's context menu — Noveller uses the same color for both. If you want a visible hover effect in the editor menu, consider also setting Menu.Highlight.

13 Scrollbars

Scrollbar tokens work differently from all other tokens. Noveller bakes opacity values directly into the brushes at load time, so you supply an opaque base color and the system applies the correct transparency automatically.

TokenOpacity appliedUsage
Scrollbar.Background0.0 (invisible)Track background at rest
Scrollbar.Background.Hover0.5Track background on hover
Scrollbar.Thumb0.5Thumb (handle) at rest
Scrollbar.Thumb.Hover0.8Thumb on hover
// Scrollbar example — use opaque colors only
"Scrollbar.Background":       "#313244",  ← becomes fully transparent
"Scrollbar.Background.Hover": "#45475a",  ← rendered at 50% opacity
"Scrollbar.Thumb":           "#cdd6f4",  ← rendered at 50% opacity
"Scrollbar.Thumb.Hover":      "#ffffff"   ← rendered at 80% opacity
.skin
Warning Any alpha channel in your scrollbar color values is ignored — the opacity listed in the table above is always what gets applied. Use #RRGGBB hex for scrollbar tokens.

14 Drop Shadow

The global drop shadow effect is controlled by a single color token. Noveller builds a DropShadowEffect with fixed blur and opacity settings — only the shadow color comes from your skin.

TokenFixed settings
DropShadowBlurRadius 12 · Opacity 0.06 · ShadowDepth 2
Tip For dark skins, set DropShadow to Black or a very dark shade. For light skins a soft warm grey like #8a7a6a tends to look more natural than pure black.

16 Minimal Example

The smallest viable skin covers just the surface, text, and accent. Noveller falls back to Colors.Black for any token you omit, so start here and expand as needed.

{
  "Colors": {
    // Surface
    "Surface.Background":  "#1e1e2e",
    "Surface.Offset":      "#181825",
    "Surface.Border":      "#313244",
    "Surface.TitleBar":    "#11111b",

    // Text
    "Text":                "#cdd6f4",
    "Text.Secondary":      "#a6adc8",

    // Accent
    "Accent":              "#cba6f7",

    // Editor
    "Editor.Background":   "#1e1e2e",
    "Editor.Text":         "#cdd6f4"
  }
}
